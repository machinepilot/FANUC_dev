============================================================
1 CLIENT OUTPUT REGISTERS (PLC → Press Brake)
============================================================

Address | Size  | Type   | Description              | Notes
--------|-------|--------|--------------------------|---------------------------
0       | 1     | WORD   | Operating Mode Selection | BIT0=Editor, BIT1=Manual, BIT2=Semi, BIT3=Auto
1       | 1     | WORD   | Command Strobe           | BIT0: 1=Execute, 0=Complete (handshake)
2       | 2     | DWORD  | Command Opcode           | 3000-4000 range; see Command Opcodes
4       | 1     | BYTE   | Data (BYTE)              | 0-255
5       | 1     | WORD   | Data (WORD)              | 0-65535
6       | 1     | WORD   | Reserved                 | Upper word for DWORD
7       | 2     | DWORD  | Data (DWORD)             | 32-bit parameter
9       | 4     | DOUBLE | Data (DOUBLE)            | Angles, dimensions; IEEE 754
53      | 40    | STRING | Data (STRING)           | Program names (80 chars max)

============================================================
4 HANDSHAKE PROTOCOL
============================================================

All commands must follow this exact sequence:

1. Write data (if needed) to appropriate register
2. Write command opcode to register 2
3. Set register 1 BIT0 = 1 (strobe)
4. Wait for register 1231 BIT0 = 1 (acknowledgment)
5. Set register 1 BIT0 = 0 (clear strobe)
6. Wait for register 1231 BIT0 = 0 (ack clear)
7. Check register 1024 for error (0 = success)

============================================================
5 IMPORTANT NOTES
============================================================

REGISTER ALIGNMENT
- DWORD (2 regs): Must start on EVEN address
- DOUBLE (4 regs): Must start on address divisible by 4
- STRING: Can start on any address

READ/WRITE LIMITS
- Max registers per read: 125 (for strings)
- Max registers per write: 123 (for strings)

REQUIRED OPERATIONS
- Heartbeat: Read any register every <10 seconds
- Mode Check: Verify operating mode before commands
- Error Check: Always read register 1024 after commands

MODE REQUIREMENTS
- Load Program: Requires EDITOR mode
- Start Cycle: Requires AUTOMATIC mode
- Modify Parameters: Requires EDITOR mode

NETWORK CONFIGURATION
- Press Brake IP: 172.16.0.51 (typical)
- PLC IP: 172.16.0.50 (typical)
- Port: 502 (standard ModbusTCP)
- Unit ID: 1

DATA FORMAT
- All multi-byte values: Big-Endian (MSB first)
- Strings: ASCII encoding, NULL terminated
- Floating point: IEEE 754 double-precision

============================================================
6 DATA TYPES
============================================================

WORD (16-bit): 1 register; 0-65535 unsigned; Big-Endian
DWORD (32-bit): 2 registers, EVEN address; Big-Endian
DOUBLE (64-bit): 4 registers, address divisible by 4; IEEE 754; Big-Endian
STRING: 2 bytes per register; ASCII; NULL terminated; max 80 chars (40 regs)

============================================================
7 USAGE EXAMPLES
============================================================

EXAMPLE 1: Load Program 'PART_A'
1a. Switch to Editor Mode: Write 1 (BIT0) to Reg 0
1b. Wait Reg 1226 BIT0=1 (Mode Ack)
1c. Clear Mode Request: Write 0 to Reg 0
1d. Wait Reg 1226 BIT0=0
2. Write Program Name to Reg 53-56
3. Write 3030 to Reg 2 (LOAD_PROG)
4. Set Strobe: Write 1 to Reg 1
5. Wait Reg 1231 BIT0=1
6. Clear Strobe: Write 0 to Reg 1
7. Wait Reg 1231 BIT0=0
8. Check Reg 1024 = 0

EXAMPLE 2: Change Angle to 95.0°
Ensure Editor Mode; convert 95.0 to IEEE 754 DOUBLE in Reg 9-12;
Write 1009 to Reg 2; strobe; wait ack; clear; verify Reg 1253-1256.

EXAMPLE 3: Change Thickness to 3.2mm
Ensure Editor Mode; convert 3.2 to IEEE 754 in Reg 9-12;
Write 1006 to Reg 2; strobe; wait ack; clear; verify Reg 1242-1245.
